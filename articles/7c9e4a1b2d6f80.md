私「わかんないよ…。

わっかんないよ！
JavaScriptの言ってることはひとつもわかんないよ！

なんで等価演算子に「==」と「===」があるの？
C言語には「==」しかないよ！ 演算子はひとつで足りてたよ！
しかもC言語の「==」は、JavaScriptの「==」と同じじゃないんだよ！
どっちかっていうと「===」の方なんだよ！
なんで見た目が同じなのに中身が違うの？
なんでイコールを増やした方が普通の挙動なの？
厳密にするなら最初から厳密にしてよ！

0 == "" がtrueってどういうこと？ 全然一緒じゃないよ！
0 == "0" もtrueってどういうこと？数値と文字列を勝手に一緒にしないでよ！

0 == "" はtrueなのに、"" == "0" はfalseって何なの！？
0 == "0" もtrueじゃん！ じゃあ "" == "0" も普通trueでしょ！？

null == undefined がtrueなのはなんで？
「無い」もの同士を勝手に仲良くさせないでよ！
[] == ![] がtrueって何！？ 自分の否定と等しいってどういう存在なの！？
NaN === NaN がfalseって何なの？ 自分自身と等しくない数って哲学なの！？

暗黙の型変換って書いて「おもてなし」って読まないでよ！
抽象等価比較って書いて「ゆるふわ比較」って読まないでよ！
ToPrimitiveとかToNumberとかvalueOfとか、裏で勝手に変換しないでよ！
変換するならちゃんと教えてよ！ 教えるならちゃんと教えてよ！

わかんないわかんないわかんないわかんない！ わかんなーい！！
JavaScriptの仕様は、昔っから何ひとつ、これっぽっちも、わかんないのよぉ！」
