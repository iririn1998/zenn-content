## `==`と`===`の違い

`===`は厳密等価演算子と呼ばれます。こちらについては皆さんご存知の通り、型と値の両方が一致するときのみ true を返します。

```js
1 === 1;     // true
1 === "1";   // false
true === 1;  // false
```

一方、`==`は等価演算子と呼ばれます。

```js
1 == "1";    // true: "1"が数値の1に変換される
true == 1;   // true: trueが数値の1に変換される
false == 0;  // true: falseが数値の0に変換される
```

私は、`==`は左右の値が falsy かどうかを基準に判定しているのだと思っていました。しかし、実際にはそうではありません。
例えば、次の式については true になります。

```js
[] == false; // true
```

直感的には、何も問題ないかと思います。

しかし、falsy な値としては [MDN](https://developer.mozilla.org/ja/docs/Glossary/Falsy) から確認することができますが、`null` / `undefined` / `false` / `NaN` / `0` / `-0` / `0n` / `""` / `document.all` です。ここに `[]` は含まれていません。

なので、truthy == falsy が true になっています。こう見ると直感的ではなくなります。
試しに、`[]`を Boolean に変換した後に比較すると `false` になります。

```js
Boolean([]); // true: 空配列はtruthy
[] == false; // true
Boolean([]) == false; // false
```

もし falsy かどうかを比べているのであれば、truthy な空配列と `false` が等しくなることはありません。
`==` の[仕様](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Operators/Equality#%E8%A7%A3%E8%AA%AC)として、左右の型が異なる場合に型変換を行うことがあることが明記されています。この型変換によって、このような不思議な（？）挙動になっています。
